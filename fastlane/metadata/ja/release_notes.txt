PeerDrop 5.3 — プッシュ通知が正しく届くようになりました。リレーサーバーの認証方式も刷新しました。

プッシュ通知の修正:v5.0~v5.2 では iOS の push entitlement が抜けていたため、アプリは APNs 登録を試みても OS に静かに拒否されており、相手から届く招待がバックグラウンドでは受信できませんでした。今回修正しました。以前通知許可を出している場合、次回起動時に自動で再登録されます。

リレーサーバー認証の強化:これまではすべての PeerDrop インストールが同じ共有 API キーを Info.plist に同梱しており、.ipa を解凍した第三者があなたを偽ってリレーサーバーを叩けてしまう状態でした。v5.3 は Apple の App Attest フレームワークに切り替え:端末が「正規の iOS インストールであること」を暗号学的に証明したうえで、サーバーが短期トークンを発行します。証明用の鍵は Secure Enclave に保存され、外には出ません。設定不要、バックグラウンドで自動的に動作します。

細かい改善:
- v5.2 で追加した「初対面の相手」確認シートが、複数の接続が同時に来ても順番に表示されるようになりました(以前は二つ目が静かにキューに溜まっていました)。
- 「近くにいる相手」タブのソートメニューが、未選択の項目に空のアイコン枠を出さなくなりました。

互換性:v5.3 と v5.0~v5.2 同士の接続はこれまで通り動作します。プロトコルバージョンは変更ありません。
