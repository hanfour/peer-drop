PeerDrop 3.4.0 — セキュリティと信頼性

セキュリティ
- 初回接触確認：相手から初めてメッセージを受け取ると、暗号化セッションを確立する前に指紋の照合を求められます——静かな中間者攻撃を防止します。
- 復号警告：同じ相手から 3 通連続で復号できない場合（通常は相手の ID 鍵が変更されたとき）、無言で失敗するのではなく、指紋の確認を促すバナーが表示されます。
- メッセージ再構築経路の強化：256 MB 上限、範囲外の chunk index 拒否、競合する重複 chunk の検出。

信頼性
- リレー接続のライフサイクルを大幅にリファクタリング。先日の v3.3.0 ホットフィックスで修正したクラスのバグは、構造上もう発生不可能になりました——リレーセッションが socket のクリーンアップを自動的に保証します。

アクセシビリティと仕上げ
- ダイナミックタイプが、発見・招待・セキュリティ系 UI で正しく拡大できるようになりました——最大のアクセシビリティ文字サイズでもクリッピングなし。
- 55 以上の UI 文字列を 5 言語に追加翻訳。

省電力
- ペットのアニメーションタイマーが、アプリのバックグラウンド時に自動停止します。

内部改善
- ワンタイム pre-key は、メモリから削除する前に「使用済み」として永続化されるようになり、ハンドシェイク中のクラッシュで再利用されることを防ぎます。
- すべての ID 鍵ローテーションは、連絡先ごとの監査トレイルに記録されます。
- ペット/ウィジェット共有状態がアトミック書き込みになり、ウィジェットが書き込み中の中途半端な状態を読み取ることがなくなりました。
