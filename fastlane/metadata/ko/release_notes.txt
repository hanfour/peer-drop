PeerDrop 5.3 — 푸시 알림이 정상적으로 도착합니다. 릴레이 서버 인증 방식도 새로 작성했습니다.

푸시 알림 수정:v5.0~v5.2에서는 iOS push entitlement가 빠져 있었기 때문에, 앱이 APNs 등록을 시도해도 OS가 조용히 거부해 다른 기기에서 보낸 초대가 백그라운드에서 도착하지 않았습니다. 이번에 수정했습니다. 이전에 알림 권한을 허용했다면 다음 실행 때 자동으로 다시 등록됩니다.

릴레이 서버 인증 강화:이전에는 모든 PeerDrop 설치가 같은 공유 API 키를 Info.plist에 함께 가지고 있어서, .ipa를 푼 사람이 누구든 사용자의 신원으로 릴레이 서버를 호출할 수 있었습니다. v5.3는 Apple의 App Attest 프레임워크로 전환합니다:기기가 "정상 iOS 설치"임을 암호학적으로 증명한 후에 서버가 짧은 토큰을 발급합니다. 증명용 키는 Secure Enclave에 저장되어 절대 기기 밖으로 나가지 않습니다. 설정할 게 없고 백그라운드에서 자동으로 작동합니다.

소소한 개선:
- v5.2에서 추가된 "처음 만나는 기기" 확인 시트가 동시에 여러 연결이 들어와도 순서대로 표시됩니다(이전엔 두 번째가 조용히 큐에 쌓였습니다).
- "주변 기기" 탭의 정렬 메뉴가 선택되지 않은 항목에 빈 아이콘 자리를 띄우지 않습니다.

호환성:v5.3와 v5.0~v5.2 사이의 연결은 그대로 작동합니다. 프로토콜 버전은 변경되지 않았습니다.
