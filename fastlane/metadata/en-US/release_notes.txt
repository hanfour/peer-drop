PeerDrop 5.2 — Local-network connections are end-to-end encrypted, and pairing with a new device now requires explicit verification.

End-to-end encryption: When two v5.2 peers connect on the same Wi-Fi, every chat message, file transfer, and signal between them is sealed inside a Double Ratchet session (the same forward-secret crypto used by Signal). Earlier versions trusted the local network and sent data in the clear once both sides shook hands — fine for a home router, less fine on a cafe AP. That's fixed.

Explicit pairing: The first time you connect to a new device, both screens show the same 6-digit verification code. Compare the codes face-to-face (or over a trusted channel like a phone call) and tap "Accept & Trust" on both sides. Until you do, files and chat are blocked — no silent first-contact trust. If the codes don't match, tap "Reject" to block the peer.

A small lock icon next to each connected peer shows the trust state: green for matched, orange for first contact (pending your approval), red triangle if the peer's identity key changed.

Backward-compatible: connecting to a v5.0.x peer still works exactly as before. Same protocol version, same data flows.

Under the hood: pet sprite rendering is now atlas-based — bundled artwork is smaller and loads faster.
