PeerDrop 3.4.0 — Security & Reliability

Security
- First-contact verification: when a peer messages you for the first time, you'll be asked to compare a fingerprint before establishing an encrypted session — preventing silent man-in-the-middle attacks.
- Decryption alerts: if PeerDrop can't decrypt three messages in a row from the same peer (usually because their identity key changed), a banner now tells you to verify their fingerprint instead of failing silently.
- Hardened the message reassembly path: caps message size at 256 MB, rejects malformed chunk indices, and detects conflicting duplicate chunks.

Reliability
- Major rework of the relay connection lifecycle. The class of bug that caused the recent v3.3.0 hotfix is now structurally impossible — the relay session enforces socket cleanup automatically, instead of relying on every code path remembering to do it.

Accessibility & polish
- Dynamic Type now scales the discovery, invite, and security UI without clipping — even at the largest accessibility text sizes.
- 55+ additional UI strings localized in 5 languages.

Battery
- Pet animation timer now pauses when the app is backgrounded.

Under the hood
- One-time pre-keys are persisted as consumed before being removed, so a crash mid-handshake can't re-expose them for replay.
- Every identity-key rotation is recorded in a per-contact audit trail.
- Pet/widget shared state now writes atomically — no more partial-state reads from the widget during a foreground update.
