## Phase 8.4 manual smoke checklist (per implementation plan)

Run on each tester device once, log results in the daily sections below.

- [ ] **Walk animation visible.** Tap-nudge or drag the floating pet → 8-frame walk cycle plays in ~1.3 s (= 8 frames / 6 fps).
- [ ] **Idle breathing visible.** Leave pet at rest for 10 s → subtle 2-frame breathing cycle plays at 2 fps.
- [ ] **Direction continuity.** Rotate pet mid-walk (toggle facingRight) → paws keep animating; no freeze on direction change. (S3 hysteresis specifically tested by oscillating velocity around 4 px/s.)
- [ ] **V5 onboarding shown once.** Fresh launch on a v4 device → V5UpgradeOnboarding sheet appears with the user's actual pet image. Tap "See It in Action" → dismisses, never re-shows.
- [ ] **Widget bridge updates.** Open widget on home/lock screen on a v4 device after first v5 launch → pet image refreshes from v4 single-frame to v5 multi-frame (idle frame 0 is fine — widget timeline refreshes per system schedule).
- [ ] **Cross-version peer.** Pair with a v4.0.1 device (if available). Bring pets together → both pets greet correctly; v5 device shows v4 peer as static, v4 device shows v5 peer as static. No crash, no missing pet.
- [ ] **No regressions in v4 features.** Onboarding flow, food, social, evolution, daily login — all behave as v4.0.1.
