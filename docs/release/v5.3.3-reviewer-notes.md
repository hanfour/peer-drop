## Re-submission for Guideline 2.1(b)

Thank you for the feedback on v5.3.2 (submission `4b1a4ef6-5c64-4167-91d6-975ce17dce78`). The reviewer noted that the In-App Purchases were not locatable in the app.

**Root cause:** the Settings → Support PeerDrop section was hidden entirely (`EmptyView`) when the StoreKit product list was empty — including the brief window before products finish loading. If the reviewer opened Settings during that window, they would see no tip cards and no indication where to find them.

**Fix in v5.3.3:** the section is now always present in Settings, with three explicit states:

1. **Loading…** with a ProgressView while StoreKit fetches products
2. **"Tap to load tip products"** retry button if the fetch returns empty or errors (e.g. weak network)
3. **3 tip cards** (☕ Coffee $1.99 / 🍱 Lunch $4.99 / 🎉 Big Thank You $9.99) once products load successfully

Reviewer can find the IAPs by:

1. Launch PeerDrop on iPad / iPhone
2. Tap the **Settings** tab (gear icon, bottom-right of the tab bar)
3. Scroll to the **"Support PeerDrop"** section (immediately above Notifications)
4. The 3 tip cards appear as soon as StoreKit returns products. If they don't (sandbox quirk), tap the visible **"Tap to load tip products"** retry button.

No other binary changes since v5.3.2. The 16 cat species×stage animation assets and v4 onboarding localization fixes from v5.3.2 are unchanged.

## What's in this update (carried from v5.3.2)

### 3 Tip Jar In-App Purchases

Three consumable products attached to this version:

- `com.hanfour.peerdrop.tip.small` — $1.99 — "a coffee"
- `com.hanfour.peerdrop.tip.medium` — $4.99 — "a lunch"
- `com.hanfour.peerdrop.tip.large` — $9.99 — "a big thank you"

Purely cosmetic gestures of appreciation; no feature gating, no subscriptions, no public donor list. PeerDrop never sees the transaction.

### Animated pets for cat / dog / bear / fox / pig

71/324 bundled species zips now contain multi-frame animation data (cat × 17, dog × 17, bear × 13, fox × 12, pig × 12). The remaining ~253 species still ship as v4 rotation-only and animate to static rotations; staged rollouts will fill them in subsequent versions.

### v4 upgrade onboarding strings

6 missing localization keys for the v3→v4 migration screen now have translations across en / zh-Hant / zh-Hans / ja / ko.

## Privacy posture (unchanged from v5.3.2)

- Apple's payment sheet collects card / Apple ID info. PeerDrop never sees it.
- No "tipped" flag persisted by us — only StoreKit knows.
- Privacy Manifest unchanged. Apple's IAP framework is excluded from "data collected" reporting.

## What didn't change

- `currentProtocolVersion` stays at 2 — v5.3.3 peers interop with v5.0–v5.3.x.
- Local-Wi-Fi end-to-end encryption + SAS pairing — unchanged.
- File transfer wire format, voice call, screenshot mode, push notifications, App Attest — unchanged.
- Privacy Manifest, age rating (4+), supply regions (174 countries) — unchanged.
