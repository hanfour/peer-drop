# v5.3.1 — App Store Reviewer Notes

**TL;DR:** Tip Jar enablement release. The `TipJarSection` UI shipped in v5.3.0 was guarded by `if products.isEmpty { EmptyView() }`. This version submits 3 consumable tip products (no feature gating, no subscriptions) alongside the unchanged v5.3 binary so they can ship together per App Store policy.

## What's new in v5.3.1

This release exists to satisfy App Store policy that the first In-App Purchase must ship with a new binary version. The v5.3 binary shipped 2026-05-14 already includes the `TipJarManager` (StoreKit 2 wrapper) and `TipJarSection` (Settings UI), but the section renders `EmptyView` until the 3 IAP products exist in App Store Connect.

### What the IAPs are

Three consumable products, all functioning as a tip jar:

- `com.hanfour.peerdrop.tip.small` — $1.99 — "a coffee"
- `com.hanfour.peerdrop.tip.medium` — $4.99 — "a lunch"
- `com.hanfour.peerdrop.tip.large` — $9.99 — "a big thank you"

### What the IAPs are NOT

- **No feature gating.** Files, chat, pets, secure channel, App Attest, push notifications — all stay free regardless of whether the user taps any tip card.
- **No subscriptions.** Tips are consumable; bought once, no expiry, no recurring charge.
- **No leaderboard / public donor list.** Tips are private. PeerDrop never sees the transaction; Apple's IAP framework handles everything.
- **No retroactive prompting.** Existing users on v5.0–v5.3 see no tip surface until they upgrade to v5.3.1 (and the Settings "Support PeerDrop" row only appears once the products are approved).

### Binary diff vs v5.3.0

Effectively identical. `project.yml`'s `MARKETING_VERSION` bumped from "5.3.0" to "5.3.1" and the Xcode project regenerated via `xcodegen`. No source changes since v5.3.0's `0e409a2` commit shipped.

## Privacy posture

- Apple's payment sheet collects the user's payment info. PeerDrop never sees it.
- No "tipped" flag persisted by us — we don't know who tipped, only StoreKit does.
- Privacy Manifest (`PrivacyInfo.xcprivacy`) unchanged. Apple's IAP framework is excluded from "data collected" reporting per Apple's own guidelines.
- Tip section renders the price in the user's local currency via StoreKit's `Product.displayPrice`; we never store or transmit those values.

## How to verify

1. Update to v5.3.1 on TestFlight.
2. Open Settings — a new "Support PeerDrop" section appears above Notifications with 3 tip cards.
3. Tap any card → Apple's standard StoreKit purchase sheet appears.
4. Cancel → no state change, no toast, App behaves identically.
5. Confirm in sandbox → "Thanks for the %@!" toast appears, no feature change anywhere in the App.
6. Repeat purchase (consumable; should succeed twice).

## What didn't change

- `currentProtocolVersion` stays at 2 — v5.3.1 peers interop with v5.0–v5.3.x / v4.x.
- Local-Wi-Fi end-to-end encryption + SAS pairing — unchanged.
- Pet system, file transfer wire format, voice call, screenshot mode, push notifications, App Attest — unchanged.
- Privacy Manifest, age rating (4+), supply regions (174 countries) — unchanged.
