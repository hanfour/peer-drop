## What's new in v5.3.3 → v5.3.4

### Pet variant rarity tiers

Each species family (cat, dog, bear, etc.) has multiple breed variants (cat-tabby, cat-bengal, cat-siamese, etc.). v5.3.4 introduces a rarity tier system that:

- Marks specific variants visually with a thin colored border around the sprite (silver for "rare", purple for "epic")
- Adjusts the chance each variant appears when a new pet hatches — rare variants drop ~1 in 9 hatches per family, epic variants ~1 in 45

Nine variants tagged in this release:

- **Rare** (silver border, ~10.9% drop within family): cat-siamese, cat-persian, dog-husky, dog-collie, bear-panda, pig-potbelly, rabbit-lionhead
- **Epic** (purple border, ~2.2% drop within family): bear-polar, fox-silver, pig-boar

Existing already-hatched pets keep their variant (Pet records pin the breed once chosen); only newly hatched pets are subject to the weighted distribution.

### Why this matters to users

The pet system was previously a flat catalog — all 5 cat breeds dropped equally, all 4 bear breeds dropped equally, etc. Adding a rarity tier turns the catalog into a collection: hatching a polar bear (epic) or panda bear (rare) is a noticeable event instead of a coin flip among siblings. The border treatment makes it readable at a glance without reading the breed name.

### Tip Jar IAPs carry over from v5.3.3

The 3 consumable IAPs from v5.3.3 (`com.hanfour.peerdrop.tip.{small,medium,large}`) are attached to this version. The Settings → Support PeerDrop section continues to work identically:

- Loading state with ProgressView while StoreKit fetches
- "Tap to load tip products" retry button if the fetch returns empty or errors
- Three tip cards (a coffee $1.99, a lunch $4.99, a big thank you $9.99) once products load

### Binary diff vs v5.3.3

Small. Three Swift source files changed:

- `Sprites/SpeciesCatalog.swift` — 9 variants gain `.rarity(...)` traits
- `Renderer/PetRendererV3.swift` — border draw between base sprite and mood overlay
- `Sprites/BodyGene+SpeciesID.swift` — weighted variant selection from seed

No new entitlements, no new resources, no new third-party SDKs. The cat / dog / bear / fox / pig / rabbit / hamster animation zips from v5.3.3 are unchanged.

## How to verify

1. Update to v5.3.4 on TestFlight.
2. Open the Pet tab. The user's current pet's species is locked at hatch time, so a fresh install (or an account migrated from v5.3.3) will likely show a common variant with no border — that's correct.
3. (Optional) Use the Settings → Hatch new pet flow (if available) and try hatching several times. A pet with the silver border = rare variant; purple border = epic variant.
4. Open Settings → Support PeerDrop. The three tip cards should appear (or show "Loading…" briefly first, or "Tap to load tip products" if StoreKit is slow). Tap to open the StoreKit purchase sheet; cancel returns Settings to its prior state.

## What didn't change

- `currentProtocolVersion` stays at 2 — v5.3.4 peers interop with v5.0–v5.3.x.
- Local-Wi-Fi end-to-end encryption + SAS pairing — unchanged.
- File transfer wire format, voice call, screenshot mode, push notifications, App Attest — unchanged.
- Privacy Manifest, age rating (4+), supply regions (174 countries) — unchanged.
