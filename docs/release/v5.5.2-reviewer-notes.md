## What's new in v5.5.1 → v5.5.2

**Connection reliability.** This maintenance release hardens the peer-to-peer connection-setup path. First-contact trust verification (the SAS security-code confirmation shown the first time two devices pair), encrypted-channel establishment, and first-message delivery are now more reliable — especially over the relay path. An internal refactor also removes a race condition in the connection read loop by giving each connection a single owning reader.

**Minor UI refinement.** When the app connects to a peer that reports it has no audio or file capability, the voice-call, voice-message, and file-transfer buttons are hidden for that conversation. This has no effect on normal device-to-device connections, where every control appears exactly as before.

## What didn't change

- End-to-end encryption (X3DH + Double Ratchet), local Wi-Fi + relay transports, SAS pairing — same protocol, just more robust setup.
- File transfer, voice calls, chat, the virtual pet, and cross-device pet sync — unchanged.
- Tip Jar IAPs (tip.small / tip.medium / tip.large), Privacy Manifest (no data collected), age rating 4+, supply regions — unchanged.
- No new features, no new entitlements, no new permission prompts, no new data collection.

## How to verify

1. Install v5.5.2 on two devices on the same Wi-Fi. They discover each other, pair with a matching SAS code, then send a file and chat — exactly as the live version does.
2. No special configuration or account is required; the app behaves identically to v5.5.1 for everyday device-to-device sharing.
