## Decision gates

### Gate 1 — End of internal soak (Day 10)

Pass to external soak if **all true:**

- [ ] 0 P0 issues open
- [ ] ≤ 1 P1 issue open AND has a mitigation plan in flight
- [ ] Migration verified on ≥ 2 distinct real-world v3.x datasets
- [ ] Cross-version peer verified once with real device pair (v4.0 ↔ v3.x)
- [ ] No widespread "pet looks wrong" feedback (>1 tester reporting same visual bug)

**If any P0 exists:** ship build 2 → reset internal soak to Day 1.

### Gate 2 — End of external soak (Day 17)

Submit for review if **all true:**

- [ ] 0 P0 issues open
- [ ] 0 P1 issues open
- [ ] iCloud round-trip verified by ≥ 1 external tester (2-device same-account)
- [ ] Crash-free rate ≥ 99% over the external soak window
- [ ] Reviewer notes (`docs/release/v4.0-reviewer-notes.md`) reviewed and pasted into ASC
- [ ] Build 1 attached to v4.0.0 in ASC

**If gates pass:**
```bash
fastlane submit_only version:4.0.0 build:1
```
(Phased rollout + manual release are current defaults — see `submit_only` lane.)
