# v5.0.2 — App Store Reviewer Notes

**TL;DR:** Bug-fix patch on top of v5.0.1. The v5 multi-frame walking animation that v5.0 advertised was silently broken — the cat's position moved across the screen but the sprite stayed on frame 0, so visually it looked like the pet was sliding. v5.0.2 fixes the animation pipeline plus three physics issues (walking into walls, floating in mid-screen instead of along the floor, hopping on landing after a drag-release). No new features, no permissions changes.

## What's new in v5.0.2

Bug-fix patch. The v5 multi-frame walking pipeline shipped in v5.0 had three independent bugs that kept the sprite animator stuck on frame 0 — pets moved across the screen kinematically but the walking sprite never cycled, so a user with a cat (50% probability) saw "my cat slides without animating." This release fixes the animation chain plus three related physics issues.

### Animation fixes

1. **Behavior provider tracks pet's species.** PetEngine init bound the provider to a random fresh-egg pet, then never rebound after the saved pet was assigned — saved cats ended up driven by random other species. Now rebinds on body change.
2. **Species-specific actions can exit.** `.scratch`/`.groom`/`.nap` etc. locked the pet there indefinitely because `nextBehavior` only handled `.idle`/`.walking`/`.thrown` transitions. Every species got the same exit clause.
3. **Animator observes engine.currentAction directly.** Was observing `physicsState.velocity` but `applyWalk` is kinematic (zeros velocity), so the pipeline never triggered the animator. Now driven by action transitions.

### Physics fixes

4. **Wall turnaround.** `applyWalk` clamped position at side walls but didn't flip `facingRight`, so pets pressed indefinitely into the wall. Now bounces back.
5. **Floor positioning.** Engine default-init y was 200 (mid-screen) because init has no screen reference. FloatingPetView snaps y to ground on appearance, and `applyWalk` continually snaps so the pet traces the floor above the tab bar.
6. **Clean landing.** Grounded pets (cat, dog, etc.) no longer bounce on drag-release. Only `physicsMode = .bouncing` species (frog, slime) bounce.

### Misc

- Evolution progress UI shows age (actual gate) instead of XP (engine never checked it).
- Swift 6 readiness cleanup: 7 strict-concurrency warnings → zero.

## Why this is shippable

No new permissions, no new APIs, no entitlement changes. Privacy Manifest unchanged from v5.0.1. All fixes are internal to rendering + physics — PetState persistence schema unchanged. ~374 pet tests pass.

## How to verify

If reviewer's pet is a cat (50% probability), they'll see leg-cycling walking animation along the bottom of the screen with proper wall turnaround. Other species show correct position movement with single-frame sprites — expected v4-fallback until Phase 3 mass-gen adds their multi-frame data. Drag-release lands cleanly without unintended hops.

## What didn't change

- `currentProtocolVersion` stays at 2 — v5.0.2 peers interop with v5.0.0 / v5.0.1 / v4.x.
- App Store Privacy Manifest, age rating (4+), data-collection profile, supply regions (174 countries) — all unchanged.
- `cat-tabby-adult` remains the only v5 multi-frame species×stage; more queued for v5.0.3.
