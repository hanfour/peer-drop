# v5.3.5 — App Store Reviewer Notes

**TL;DR:** Bug fix for the 92 animated pet zips. Walk animations were silently dropped during asset normalization in v5.3.3/v5.3.4 builds — pets played their idle cycle even when their state transitioned to walking. v5.3.5 ships re-normalized zip assets so the walk frames are actually present and play. No new features, no entitlements, no permission changes.

## What's new in v5.3.4 → v5.3.5

### Walk animations restored for 92 pet species

v5.3.3 introduced multi-frame animation data for 92 cat / dog / bear / fox / pig / rabbit / hamster species×stages (each species has both walk and idle frame sequences). However, the asset normalization step in our build pipeline used a frame-count heuristic that misclassified the walk frames as idle and silently dropped them. The bundled zips ended up with only an idle slot — so pets played the idle cycle even when their state transitioned to walking.

v5.3.5 re-normalizes all 92 zips with the correct walk/idle slot tagging. Users on v5.3.4 see their pets idling in place; users on v5.3.5 see them actually walking when they move across the view.

### Tip Jar IAPs carry over unchanged

The 3 Tip Jar IAPs from v5.3.3 continue to work identically in Settings → Support PeerDrop. No re-attach needed.

### Rarity tier border treatment carries over from v5.3.4

Tagged variants still render with their silver/purple border. Weighted hatch distribution unchanged.

### Binary diff vs v5.3.4

Small. Two Python build scripts changed (don't ship in binary) plus 91 zip assets re-normalized in place — same files, fixed metadata.

No new entitlements, no new resources beyond re-normalized zips, no new third-party SDKs.

## How to verify

1. Update to v5.3.5 on TestFlight.
2. Open the Pet tab. The pet should walk visibly (legs lift, body bob) when its state is walking.
3. Compare with v5.3.4 on a second device: pet stays in place playing idle.

## What didn't change

- `currentProtocolVersion` stays at 2 — v5.3.5 peers interop with v5.0–v5.3.x.
- Local-Wi-Fi end-to-end encryption + SAS pairing — unchanged.
- File transfer wire format, voice call, screenshot mode, push notifications, App Attest — unchanged.
- Privacy Manifest, age rating (4+), supply regions (174 countries) — unchanged.
