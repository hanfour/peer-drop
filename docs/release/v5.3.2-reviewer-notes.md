## What's new in v5.3.1 в†’ v5.3.2

### 3 Tip Jar In-App Purchases unlock

v5.3 shipped the `TipJarSection` UI in Settings but it was gated by `if products.isEmpty { EmptyView() }`. With the 3 consumable products approved separately, this release attaches them to the version submission so the section now renders 3 tip cards in Settings в†’ Support PeerDrop:

- `com.hanfour.peerdrop.tip.medium` вЂ” $4.99
- `com.hanfour.peerdrop.tip.large` вЂ” $9.99

These are **purely cosmetic gestures of appreciation**:

- **No feature gating.** Files, chat, pets, secure channel, App Attest, push notifications вЂ” all stay free regardless of whether the user taps any tip card.
- **No subscriptions.** Consumable products; bought once, no recurring charge.
- **No public donor list.** PeerDrop never sees the transaction; Apple's IAP framework handles everything.
- **No retroactive prompting.** Existing users see the new section only after upgrading to v5.3.2.

### v4 upgrade onboarding missing translations

V3.x users migrating to v4+ see a one-time onboarding screen (`V4UpgradeOnboarding.swift`) that listed 7 localized strings. 6 of those keys were referenced in code but absent from `Localizable.xcstrings`, so the screen displayed raw keys like `v4_upgrade_title` instead of the translated text. Added the missing 6 keys Г— 5 languages (en/zh-Hant/zh-Hans/ja/ko).
