# v5.1 ‚Äî App Store Reviewer Notes

**TL;DR:** Security-foundation release. Adds end-to-end encryption (Double Ratchet, the protocol Signal uses) on top of every local-Wi-Fi peer connection. No new entitlements, no new permissions, no new data collection ‚Äî the crypto runs purely between the two devices on the local network. Backward-compatible: a v5.1 device connecting to a v5.0.x device transparently downgrades to the prior plaintext-after-handshake behavior.

## What's new in v5.1

Adds end-to-end encryption to local-Wi-Fi peer connections. PeerDrop previously authenticated peers with a Curve25519 identity exchange during the hello handshake, but business data (chat, files, voice signaling) flowed in cleartext after the TCP socket came up ‚Äî fine on a trusted home network, weak on a shared AP. v5.1 wraps every non-control PeerMessage in a Double Ratchet session (same construction as Signal: X3DH-style bootstrap, per-message forward-secret keys).

### Wire-protocol additions

- **`PeerIdentity.supportsSecureChannel: Bool`** ‚Äî capability flag in the hello payload. v5.0.x peers decode it as `false` via a custom Codable decoder, so they're never sent a `.secureHandshake` they couldn't parse.
- **`MessageType.secureHandshake`** ‚Äî exchanges 32-byte ratchet public keys + 32-byte identity public keys. Sent in plaintext (no choice; it bootstraps the channel).
- **`MessageType.secureEnvelope`** ‚Äî wraps the ciphertext + ratchet metadata for every business message after the handshake completes. The inner plaintext is a normal `PeerMessage`.

### Protocol behavior

- The handshake is fully automatic ‚Äî kicks off the moment both sides finish the hello exchange and the capability flag confirms support. The user does not interact with it.
- Both sides initiate concurrently; lexicographic order of public keys decides which side becomes the ratchet initiator. Duplicate handshakes are dropped if the channel is already `.secured` ‚Äî replay-safe.
- If the handshake doesn't complete within 5 seconds, the connection falls back to `.fallbackPlaintext` rather than hanging forever. This is the same security level as v5.0.x.
- `.ping`/`.pong` bypass encryption (keepalive is cheap and contains no sensitive data).

### Pinning UI

Connected-peers list shows a small icon next to each peer:

- `lock.fill` (green) ‚Äî peer's identity key matches a previously trusted contact.
- `lock.shield` (orange) ‚Äî first time meeting this peer (TOFU; trust-on-first-use).
- `exclamationmark.triangle.fill` (red) ‚Äî peer's identity key changed from what was previously stored. Existing `KeyChangeAlertInfo` flow already handles the user prompt.
- `lock.open` (gray) ‚Äî handshake didn't complete; channel is plaintext (peer is v5.0.x or didn't respond in time).

## Why this is shippable

- No new permissions or entitlements. Encryption keys live in the existing `IdentityKeyManager.shared` (Curve25519, derived from app installation, already on disk in v5.0.x). Privacy Manifest unchanged.
- No new data collection. Keys are local-only and never leave the device.
- Backward-compatible: a v5.1 device talking to a v5.0.x device sees `supportsSecureChannel: false` and stays on the plaintext path ‚Äî connections to existing users keep working.
- Pure local computation. No network calls outside the existing local-TCP / Cloudflare-relay flows, both of which v5.0.x already shipped.

## How to verify

1. Connect two v5.1 devices on the same Wi-Fi. Within ~1 second, the green `lock.fill` icon should appear next to each peer's name.
2. Connect a v5.1 device to a v5.0.x device. No icon appears (channel stays plaintext, exactly as v5.0.x behavior).
3. Send a chat message after the lock appears. The on-wire frame type is `.secureEnvelope` (verifiable in Console logs); the plaintext string never appears in the network payload.

## What didn't change

- `currentProtocolVersion` stays at 2 ‚Äî v5.1 peers interop with v5.0.x / v4.x.
- Pet system, file transfer, voice call, screenshot mode ‚Äî no behavior changes.
- App Store Privacy Manifest, age rating (4+), data-collection profile, supply regions (174 countries) ‚Äî all unchanged.
- `cat-tabby-adult` remains the only v5 multi-frame species√óstage; more queued.
