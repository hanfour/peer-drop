# v5.0.x Weekly Cadence Playbook

Phase 3 mass-gen is a marathon (~5 months at the 2,000/mo PixelLab quota for full
33-species × 3-stages coverage). Rather than wait for full coverage to ship,
v5 adopts a **weekly minor-version asset-update cadence**. Each release adds
a handful of newly-generated species, ships fast, and rebalances the genome
distribution toward already-animated species.

## Weekly cycle

| Day | Activity | Estimated time |
|---|---|---|
| Mon–Tue | PixelLab generation: 2–4 species×stage at 68×68 | 2–3 hr operator |
| Wed | Run `Scripts/normalize-pixellab-zip.sh` per zip; drop into `PeerDrop/Resources/Pets/`; update `expectedV5Coverage` whitelist | 30 min |
| Thu | Run full test sweep + manual smoke on simulator | 30 min |
| Thu | Bump `MARKETING_VERSION` patch (5.0.0 → 5.0.1, 5.0.1 → 5.0.2…); update 5-lang release notes; commit | 20 min |
| Thu | `fastlane release` — submit | 30 min |
| Fri | Apple Store review (typically &lt;24 hr for resource-only updates) | passive |

## Why weekly works

- **No new code** between v5.0.0 and v5.0.x. Just asset additions + `expectedV5Coverage` updates + version bumps. App Store review treats this as "metadata + bundle resources" — historically &lt;24 hr review for PeerDrop's prior cadence (v3.x rolled out 9 versions in 14 days).
- **Genome redistribution** keeps "v5-animated species probability" high as more species reach v5. Each version's release notes can adjust `PetGenome.from()` weights to favor the latest-animated species during its rollout window.
- **Operator cadence sustainable.** ~3 hr/week is ~12 hr/month — fits around any other workload.

## Per-week deliverable shape

A representative v5.0.x release ships:

- **2–4 new species at v5 schema** (e.g. `dog-shiba-adult.zip`, `slime-green-adult.zip`, `dragon-western-adult.zip`)
- **`expectedV5Coverage`** whitelist updated with 6–12 new entries (3 stages × 2–4 species)
- **Release notes** mention the species joining v5 animation (e.g. "Dogs and slimes now walk with multi-frame animations")
- **`PetGenome.from()` weights** rebalanced: ↑ for newly-animated, ↓ for still-pending

## Release notes template (per minor)

```
PeerDrop 5.0.X — More Pets Animated

What's new:
- [Species names] gained multi-frame walk animations.
- Subtle improvements to existing animation cooldowns.

Coming soon:
- More species joining the animation rollout each week.
```

## When to bump minor vs patch

- **5.0.X (patch):** Asset additions only. No code changes. Most weekly releases.
- **5.1.0 (minor):** Code changes — new actions (sleep, eat), new behavior tunings, UI tweaks. Ship when accumulated code changes warrant a roundtrip through review.
- **6.0.0 (major):** Architectural rewrite (e.g. sprite atlas system, behavior JSON). Roadmap deferral document at `docs/plans/v5.1+-deferred.md` outlines candidates.

## Phase 3 acceptance gate handoff

When `expectedV5Coverage` covers every multi-variety species × 3 stages
(roughly 96 entries), flip `phase3Complete = true` in
`PeerDropTests/Pet/MainBundleAssetCoverageTests.swift`. The acceptance
gate test un-skips and enforces the contract going forward. At that
point:

1. Drop the cat-family genome bias from `PetGenome.from()` (re-balance to uniform).
2. Update reviewer notes to drop the "Initial coverage focuses on common species" caveat.
3. Final 5.X.0 release notes can claim full coverage.

## Tracking progress

The authoritative "what's at v5" source is `expectedV5Coverage` in
`MainBundleAssetCoverageTests.swift`. Don't duplicate the list elsewhere
— that always drifts. CI badges in README (item #9 in the v5.1+
deferred design doc) will surface this externally once the GitHub
Actions workflow lands.

## Risks + mitigations

| Risk | Mitigation |
|---|---|
| Apple flags weekly cadence as "spam" | Each release has a clear value-add (new animated species). PeerDrop's history of weekly releases (v3.x cycle) wasn't flagged. |
| Quota depletion mid-month | Generate Mon–Tue early in the month; prioritize most-popular species (cat variants, dog, slime). |
| Genome-bias drift causing user surprise | Log redistribution in release notes; commit-to-commit weight changes are small (5–10 pp adjustments). |
| Operator burnout | Stop cadence at any time without breaking anything — partial coverage already ships safely. Cadence is a goal, not a contract. |

## Pre-cadence checklist (before week 1)

- [ ] v5.0.0 submitted to App Store (this happens via `docs/release/v5.0-submission-runbook.md`)
- [ ] v5.0.0 approved + live (Apple typically &lt;48 hr first-time major)
- [ ] PixelLab Apprentice subscription active + sufficient monthly quota
- [ ] Operator has 2–3 hr/week reserved for generation
- [ ] `Scripts/test_normalize_pixellab.py` passes locally — `python3 -m unittest Scripts.test_normalize_pixellab`

## Per-cycle execution (week N)

- [ ] Pick species batch (~2–4 species×stages, prefer common/popular)
- [ ] Generate via PixelLab at 68×68
- [ ] Normalize each: `Scripts/normalize-pixellab-zip.sh raw.zip <species>-<stage>.zip`
- [ ] Drop into `PeerDrop/Resources/Pets/`
- [ ] Update `expectedV5Coverage` in `MainBundleAssetCoverageTests.swift`
- [ ] Run `xcodebuild test -only-testing:PeerDropTests/MainBundleAssetCoverageTests` — passes
- [ ] Run full sweep `xcodebuild test` — passes
- [ ] Optional: tune `PetGenome.from()` weights to favor newly-animated species
- [ ] Bump `MARKETING_VERSION` (5.0.X → 5.0.X+1)
- [ ] Update 5-lang `fastlane/metadata/*/release_notes.txt` with the new species
- [ ] Update reviewer notes if needed (`docs/release/v5.0-reviewer-notes.md` or version-specific)
- [ ] Commit on `main` (or via PR for review): `release(v5.0.X+1): added <species>`
- [ ] `fastlane release` from repo root

## Timeline projection

Assuming 3 species × 3 stages per week, Phase 3 acceptance gate can flip in:

```
33 species − 2 already done (cat-tabby, ghost) = 31 remaining
31 species × 3 stages = 93 species×stage entries
At 9 entries/week (3 species × 3 stages): 93 / 9 ≈ 11 weeks
At 6 entries/week (2 species × 3 stages): 93 / 6 ≈ 15 weeks
```

Realistic range: **3–4 months** to full v5 coverage at sustainable pace.
Faster sprints possible during quota-rich months but not necessary.
