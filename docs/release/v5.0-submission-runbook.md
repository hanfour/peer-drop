## Run the release

```bash
cd "/Volumes/SATECHI DISK Media/UserFolders/Projects/applications/peer-drop"
fastlane release
```

This single command:

1. Reads `MARKETING_VERSION` from `project.yml` → detects `5.0.0`
2. Loads reviewer notes from `docs/release/v5.0-reviewer-notes.md` BEGIN_PASTE/END_PASTE block
3. Builds Release config of PeerDrop scheme
4. Uploads to TestFlight (build 1 — first build for this version)
5. Submits for App Store review with auto-release on approval, no phased rollout
6. Reviewer notes auto-pasted into ASC App Review Notes field

**Expected wall time:** 15–30 minutes (build + upload + submission API roundtrip).

## What's in this submission

- v5 multi-frame sprite animation pipeline (parser, service cache, renderer overload, animator, engine integration)
- Aging fix: adult → elder requires age ≥ 90 days AND interacted within last 30 days (was 14 days age-only). Migration demotes incorrectly-elder pets back to adult on first v5 launch.
- V5UpgradeOnboarding: shown once to existing v4 users on first v5 launch
- Widget bridge invalidation on first v5 launch (rendered image refreshes)

- Pets stuck as elder forever after 2 weeks → fixed

- Ghost flicker/vanish too rare → cooldowns tightened (5–10s and 60–90s)

- `ghost.zip`: south walk + south idle. New 68×68 translucent ghost.
- All other 32 species × 3 stages remain at v4 schema. SpriteService gracefully degrades to 1-frame static rendering — no broken pets.

## If something goes wrong

### Build fails (signing)
Most likely cause: Apple Developer account session expired in Xcode.

```bash
open -a Xcode
# Xcode > Settings > Accounts > re-sign in to Apple ID
# Confirm "PeerDrop" team is selected and provisioning is current
```

### Upload to TestFlight fails
Check `fastlane/api_key.json` exists and has correct issuer/key contents.

```bash
ls -la fastlane/api_key.json
# If missing, regenerate from App Store Connect → Users and Access → Keys
# Save as fastlane/api_key.json (gitignored).
```

### Reviewer notes not loaded
Check the BEGIN_PASTE/END_PASTE markers in `docs/release/v5.0-reviewer-notes.md` are intact. Lane prints the loaded char count — should be ~2,355.

### Submission rejected by Apple

Most likely flag: "advertised features not visible during review" if reviewer's randomly-generated pet happens to be a non-cat species (sees v4 static instead of v5 walk animation). Mitigations:

- Default genome creation favors `cat` family → reviewer's fresh-install pet has high probability of being cat-tabby. New release notes wording avoids overclaiming.
- Reviewer notes §1 honestly describes "render-side upgrade with Initial coverage" framing.
- If rejected on this basis: respond with a screencast of the cat-tabby pet walking, since the feature IS implemented.

### Need to abort

```bash
# If submission already created on ASC:
fastlane check_status  # see current state
# Then manually: ASC → My Apps → PeerDrop → 5.0 → Reject Binary
```
