# v5.0.1 — App Store Reviewer Notes

**TL;DR:** Removes the Ghost species (visual quality issue) and silently migrates existing Ghost-bodied pets to a Cat appearance, preserving every aspect of the pet's identity (name, age, level, interaction history). No new permissions, no privacy-relevant changes, no features added. The fastlane `release` lane copies the BEGIN_PASTE/END_PASTE block below into App Store Connect's "Notes for App Review" field.

## What's new in v5.0.1

A render-side patch on top of v5.0.0:

1. **Removes the Ghost species.** Ghost's translucent visual never reached an acceptable quality bar across our regeneration attempts. Rather than ship more partial fixes, we retired the species entirely. Existing Ghost-bodied pets — about 10% of v5.0.0 fresh installs by genome distribution — silently migrate to a Cat appearance on first launch of v5.0.1. The pet's identity is fully preserved: same UUID, same display name, same birth date, same age, same evolution level, same interaction history. Only the rendered sprite changes.
2. **Genome distribution rebalanced.** The 10% Ghost band collapses into Cat (now 50%). Other species weights unchanged. Tracked in `BodyGene.from(personalityGene:)`.
3. **Migration safety net.** `BodyGene.init(from:)` decoder maps the legacy `body: "ghost"` JSON value to `.cat` silently. `PetEngine.migrateGhostBodyForV501()` runs once per device on first v5.0.1 launch, force-persisting the corrected body so it lands on disk immediately rather than waiting for the next backgrounding event. Both behaviors covered by `GhostMigrationTests`.

## Why this is shippable

- **No data loss.** Ghost-owners keep their pet — only the species visual changes.
- **No new permissions, no privacy-relevant changes.** Same as v5.0.0 baseline (Privacy Manifest unchanged).
- **No new APIs, no new entitlements.** Pure code/asset deletion + decoder shim + one-shot migration.
- **No regression to v5.0.0 features.** v5 multi-frame walk + idle animations remain intact for cat-tabby-adult (the polished asset). Other species continue to render as v4-style 1-frame static via `SpriteService` C1 fallback.

## How to verify

If reviewer's randomly-generated pet ends up as a Cat, that's the expected v5.0.1 distribution. If you want to test the migration path:

1. There's no UI surface for selecting Ghost specifically — the genome assigns species probabilistically. Any pet at `body: "cat"` after v5.0.1 install is the post-migration state.
2. The `Settings → Pet → Reset` flow generates a fresh pet from the rebalanced distribution. Cat (50%), Dog (10%), Rabbit (8%), Bird (8%), Frog (6%), Bear (6%), Dragon (4%), Octopus (4%), Slime (4%).
3. Console.app `subsystem == "com.hanfour.peerdrop" && category == "PetEngine"` shows the migration log line on first launch:
   `v5.0.1 ghost migration: persisted current pet state — body=cat`

## What didn't change

- Cross-version peer payload compatibility: `currentProtocolVersion` stays at 2. v5.0.1 peers transparently interop with v5.0.0 / v4.x peers — the body field still encodes as a string and any peer receiving `body: "ghost"` from a yet-to-update v5.0.0 sender's payload decodes via the same migration path.
- Test coverage: ~960 unit tests still pass; ghost-specific tests (≈18) deleted, 6 new `GhostMigrationTests` added.
- App Store Privacy Manifest, age rating (4+), data-collection profile, supply-region list (174 countries) — all unchanged from v5.0.0.

## Build details

- Build 1, Release configuration, manual signing
- Cat-tabby-adult retains v5 multi-frame coverage from v5.0.0 (south + east + west walk, south idle)
- Bundle size reduced by ~22 KB (`ghost.zip` removed)
