## 1. Goal

Bring the iOS production app's pet system up to the visual + interaction standard validated in the web prototype (`docs/pet-design/web-prototype/`):

- **Visual**: Vadimsadovski-style minimalist pixel sprites at 48×48 (replacing 16×16 hand-coded)
- **Lifecycle**: 3 stages per species (幼體 / 成熟體 / 老年體) — replacing current 2 stages (`baby` / `child`)
- **Patterns**: 5 multi-color seed-driven patterns (replacing current 2 single-color patterns)
- **Identity**: Each pet has a stable `seed` driving pattern layout — same pet always looks the same
- **Species expansion**: 10 → 11 species (add 龍貓 totoro alongside octopus, both retained)

**Constraints**:
- No PixelLab subscription (free tier 5/day daily slow generations only)
- Existing user pets must not break — full backward compatibility
- Visual change is opt-in via Settings (Classic Mode toggle preserved through all v4.x)

## 2. Scope

### In scope (v4.0)

- New 3-stage `PetLevel` enum + persistence migration

- Multi-direction sprite usage in main UI (production stays side-view + flip; 8-direction available for future)
- Sound design
- New species beyond totoro

## 3. Architecture

### 3.1 PetLevel + persistence migration

**Current**:
```swift
enum PetLevel { case egg, baby, child }
```

**New**:
```swift
enum PetLevel: String, Codable {
    case egg
    case baby      // 0-7 days
    case adult     // 8+ days (was `child`)
    case elder     // 14+ days (new)
}
```

**Migration on first v4.0 launch**:
1. Existing `child` raw value → decode as `adult` (rawValue identical, just renamed; `child` kept as Codable alias)
2. For each pet: if `Date() - hatchedAt > 14 days` AND current stage is `adult`, auto-promote to `elder`. Track in `migrationDoneAt` to avoid re-promotion.
3. New pets: `baby` → `adult` at day 8, `adult` → `elder` at day 14. (Existing baby→child timing was 7 days; we adjust to 8 for new boundary.)

### 3.2 BodyMeta + per-species data

**Current**: each species (e.g., `CatSpriteData`) has `baby` and `child` static dictionaries.

**New**: each species has `baby`, `adult`, `elder` dictionaries. Bird is special (adult is `rooster` or `hen` sub-variant). Ghost has only `adult`.

`BodyMeta` extended:
```swift
struct BodyMeta {
    let eyeAnchor: (x: Int, y: Int)
    let patternMask: [[Bool]]
    let groundY: Int           // legacy 16×16 ground offset
    let groundOffsetsY: [PetLevel: Int]  // new — per-stage feet position in 48×48 frame
    let displayScales: [PetLevel: Double]  // new — per-stage size (cat baby=0.7, octopus elder=1.15)
    let hangAnchor: (x: Int, y: Int)
    let climbOffset: (x: Int, y: Int)
}
```

`SpriteDataRegistry` updated:
```swift
enum SpriteDataRegistry {
    static func sprites(for body: BodyGene, stage: PetLevel) -> [PetAction: [[[UInt8]]]]?
    // new: returns 3 stages instead of 2
}
```

### 3.3 Bird sub-variants

Bird has `baby` (chick), `adult-rooster`, `adult-hen`, `elder`. New BirdGene field:
```swift
struct BirdGene {
    let adultVariant: BirdVariant  // .rooster | .hen — randomly assigned at hatch
}
```
Existing bird pets: assign random variant on migration.

### 3.4 New octopus + totoro

```swift
enum BodyGene {
    case cat, dog, rabbit, bird, frog, bear, dragon, octopus, ghost, slime
    case totoro  // NEW
}
```

`OctopusSpriteData` retained with all v3.x action data.
`TotoroSpriteData` added with v4.0 stages.

`PetGenome.randomBody()` adjusted: `octopus` removed from random rotation for new pets (so new pets pick from 10 species: cat/dog/rabbit/bird/frog/bear/dragon/slime/ghost/totoro). Octopus retained for existing pets only.

### 3.5 Pattern system upgrade

**Current**: `PatternGene { stripe, spot }`. Single color — applies palette slot 6 to mask pixels.

**New**: `PatternGene { plain, stripe, spot, twoTone, star }`. Multi-color — picks 1-3 slots from {3, 4, 5, 6} based on `seed`.

Migration: existing pets with `.stripe` or `.spot` retain pattern, but layout becomes seed-driven (so will look slightly different but in the same family). Pets without pattern get `.plain`.

### 3.6 Per-pet seed

```swift
struct PetGenome {
    // ... existing fields
    let seed: UInt32  // NEW — used by pattern overlay
}
```

Migration: existing pets get `seed = hash(petId + petName)` so layout is stable + unique per pet.

## 4. Visual + scope-of-art

### 4.1 What gets generated

For v4.0, generate the **minimum viable action set** per species×stage:

| Action | Why | Frames |
|---|---|---|
| idle | always-on default | 4 (existing 8-dir static can be frame 0) |
| walking | locomotion + transfer | 4 (NEW PixelLab gen) |
| sleeping | sleep state | 2 (frame 0 = idle, frame 1 = curled) |
| happy | success / petted | 2 (NEW or reuse static + animation) |
| scared | fail | 2 (NEW or reuse static) |

Other actions (run, jump, climb, hang, fall, sitEdge, eat, yawn, poop, angry, love, tapReact, pickedUp, thrown, petted) — defer to v4.1+ via reuse of `idle` until upgraded.

### 4.2 PixelLab generation budget

We have 27 lifecycle character bases already generated (8-direction static rotations) in `docs/pet-design/ai-brief/species-zips-stages/` + `species-zips/`. These directly become production assets.

**New generations needed for v4.0**:
- Walking 4-frame per species×stage = 27 × 1 = **27 generations**
- Totoro baby + adult + elder static = **3 generations**
- Bird elder rooster + hen variants animated walk = covered above (rooster/hen are adult)
- Total: **30 generations**

**Free tier**: 5 daily slow generations
**Estimated time**: 30 ÷ 5 = **6 days** of consecutive daily generation

Plus ~1-2 days for any retries / fixes. **Realistic: 7-10 days**.

### 4.3 Sprite quality acceptance

Each generated sprite must:
- Match Vadimsadovski minimalist style (consistent with prototype v3)
- Have clear silhouette (foreground readable from common backgrounds)
- Clean per-direction (8-rotation set must visually feel like same character)
- Walking 4-frames must read as locomotion (legs alternate visibly)

Failed generations get one re-prompt; if still bad, fall back to the 8-dir static (no animation) and flag for v4.1.

## 5. Onboarding UX

### 5.1 First launch after v4.0 install

**Splash modal** (skippable, shown once):

- Title: 「你的寵物升級了 🌱」
- Body: side-by-side comparison of user's actual pet (current 16×16 vs new 48×48)
- 3 bullets:
  - ✨ 全新像素風格、更細緻
  - 👶👴 加入幼體、成熟體、老年體三階段
  - 🎨 五種花紋與多色配色組合
- Two buttons:
  - 「太棒了，看看新樣子」 → continues to main app with v4 enabled
  - 「先用回原本樣子」 → enables Classic Mode flag, continues to main app with v3.x rendering

### 5.2 Background data migration

Triggered automatically on first v4.0 launch:

1. Decode existing `PetState` with `child` raw value as `.adult`.
2. For each pet where `Date() - hatchedAt > 14 days`:
   - If current stage is `.adult`, promote to `.elder`.
   - Set `migrationDoneAt = Date()`.
3. For each pet, generate `seed = hash(petId + petName)` if not yet set.
4. Bird pets: assign random `adultVariant` (rooster | hen) if not yet set.
5. Octopus pets: NO change — retain octopus species. Show octopus → totoro info in step 5.4.

### 5.3 First-time spotlight (after splash, after main UI loads)

If user did NOT enable Classic Mode:
- Highlight the new lifecycle tab/section in pet detail UI
- Spotlight tooltip: "現在你的寵物有 3 個生命階段，點這裡切換查看"
- Single-tap dismiss; one-time

### 5.4 Octopus → totoro notification (if user has octopus)

If any of user's pets is `.octopus`:
- After main UI loads, show one-time dialog:
  - Title: 「新物種：龍貓 🌲」
  - Body: 「你的章魚仍然是章魚！但 PeerDrop 新增了「龍貓」物種。新養的寵物可以選擇龍貓作為新成員。」
  - Buttons: 「了解」（dismiss）/「看看龍貓樣子」（opens species picker preview）

User keeps existing octopus pet untouched. Future newly-hatched pets pick randomly from 10 species (octopus excluded).

### 5.5 Settings → Pet → Classic Mode

- Toggle: 「使用經典 16×16 風格」
- Default: OFF (v4 style)
- When ON: renderer uses v3.x sprite data path; user's pet stage stays at current promoted level (no elder visible since elder doesn't exist in v3.x)
- Affects: main pet display, Lock Screen widget, all pet-related UI surfaces
- **Preserved through entire v4.x lifecycle** (all minor versions retain this toggle)

## 6. Implementation phasing

| Phase | Scope | Time | Generates |
|---|---|---|---|
| **A — Infrastructure** (no PixelLab) | PetLevel + BodyMeta + Renderer 16×16→48×48 + Pattern upgrade + seed field + BodyGene.totoro | 2 weeks | nothing |
| **B — Sprite generation** (PixelLab daily slow) | 30 generations spread over ~7-10 days | 1.5 weeks | 30 sprites |
| **C — UI + Onboarding** (no PixelLab) | Lifecycle picker UI in iOS, settings toggle, onboarding flow, octopus → totoro dialog | 1 week | nothing |
| **D — Persistence migration + tests** | Migration code, 292 test updates, new lifecycle / pattern tests, Lock Screen widget | 1 week | nothing |
| **E — Release** | TestFlight beta, PR review, 5-language translation review, App Store submission | 1 week | nothing |

**Total**: 5.5–7 weeks

**Critical path**: Sprite generation (Phase B) is the bottleneck due to free-tier daily limit. Phase A and Phase C work can proceed in parallel during Phase B's 7-10 day generation period.

## 7. Risk + mitigations

| Risk | Mitigation |
|---|---|
| User dislikes new style | Classic Mode toggle preserves 16×16 path indefinitely |
| Daily generation hits failures (queue stuck like frog elder did) | Retry next day on free quota; sprite fallback to 8-dir static if walk gen fails |
| 48×48 sprite assets noticeably increase app bundle size | Compress + lazy-load per species (only load sprites for owned pets); profile before release |
| Existing pet save data corruption | Codable alias (`child` → `adult`) tested; fallback: if migration fails, use Classic Mode automatically |
| Pattern visual change on existing pets | Same `PatternGene` enum value gives same visual category; layout shifts to seed-driven but reads as "same pattern, slight refresh" |
| Bird sub-variant random assignment on migration | Show user a "你的鳥已決定為公雞 / 母雞" notification on first interaction |
| Octopus pets stay as octopus forever | Documented; user told they can manually rename or evolve via gene-edit feature (future) |

## 9. Success criteria

- Existing pets render correctly in new style after v4.0 install (no broken sprites, palette intact)
- Pet ages > 14 days appear as elder on first v4.0 launch
- 5 patterns × 8 palettes × 3 stages × per-pet seed = visible variety across users
- Classic Mode toggle works for both main UI and Lock Screen widget
- 292 tests + new tests all pass
- App bundle size increase < 8MB
- TestFlight beta receives positive feedback from > 60% of testers
