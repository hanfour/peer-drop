# v3.0 Phase 2.5 — End-to-End Remote Communication Integration

## Goal

Wire the Signal Protocol building blocks (X3DH, Double Ratchet, PreKeyStore, MailboxClient, Cloudflare Worker v2) into a working end-to-end remote communication flow. Users can generate invite links, establish encrypted sessions, and exchange messages — all using the existing chat UI with minimal new screens.

## Approach: Hybrid (Online relay + Offline mailbox)

- **Online:** Messages delivered instantly via mailbox relay polling
- **Offline:** Messages stored in Cloudflare KV (7-day TTL), pulled on next app launch
- **UI:** Minimal — add "Send Message" to existing SecurityDashboard trusted contacts; reuse ChatView

## Architecture

### 1. Invite Link Receive Flow

```
Carol opens peerdrop://invite?mbx=...&fp=...&name=...&exp=...
  → PeerDropApp .onOpenURL handler parses InvitePayload
  → Validate expiry
  → Show confirmation sheet (InviteAcceptView): "Alice wants to connect. Accept?"
  → Accept → MailboxManager.registerIfNeeded()
  → fetchPreKeyBundle(aliceMailboxId)
  → X3DH initiator handshake
  → Add Alice to TrustedContactStore (trust: .linked, mailboxId set)
  → Encrypt X3DH initial payload → send to Alice's mailbox
  → Alice polls → receives initial message → responder handshake → session established
```

**Files:** Modify `PeerDropApp.swift`, create `PeerDrop/UI/Security/InviteAcceptView.swift`

### 2. Remote Message Envelope

```swift
struct RemoteMessageEnvelope: Codable {
    enum EnvelopeType: String, Codable {
        case x3dhInitial   // Session establishment
        case message       // Encrypted chat message
    }
    let type: EnvelopeType
    let senderMailboxId: String
    let ratchetMessage: RatchetMessage?       // for .message
    let x3dhPayload: X3DHInitialPayload?      // for .x3dhInitial
}

struct X3DHInitialPayload: Codable {
    let identityKey: Data
    let ephemeralKey: Data
    let signedPreKeyId: UInt32
    let oneTimePreKeyId: UInt32?
    let senderDisplayName: String
    let senderFingerprint: String
    let initialCiphertext: Data  // First message encrypted with the new session
}
```

**Files:** Modify `PeerDrop/Security/Protocol/RemoteMessageEnvelope.swift`

### 3. Message Routing (handleRemoteMessage)

```
MailboxManager.onMessageReceived → MailboxMessage
  → Base64 decode ciphertext
  → JSON decode → RemoteMessageEnvelope
  → switch envelope.type:
    case .x3dhInitial:
      → Look up or create contact by senderMailboxId
      → RemoteSessionManager.respondToSession(...)
      → Update TrustedContactStore (trust: .linked)
      → Decrypt initial ciphertext → route to ChatManager
    case .message:
      → Find contact by senderMailboxId
      → RemoteSessionManager.decrypt(message, from: contactId)
      → Route to ChatManager.handleRemoteChat(text, from: contactId)
```

**Files:** Modify `ConnectionManager.swift` (handleRemoteMessage)

### 4. Sending Remote Messages

```
SecurityDashboardView → trusted contact row → "Send Message" button
  → Open ChatView (using contact UUID as peer identifier)
  → User types message → ChatManager.sendRemoteMessage(text, contactId)
    → RemoteSessionManager.encrypt(data, for: contactId)
    → Wrap in RemoteMessageEnvelope(.message)
    → ProofOfWork.generate(challenge)
    → MailboxClient.sendMessage(to: peerMailboxId, ciphertext, pow)
```

**Files:** Modify `SecurityDashboardView.swift`, `ChatManager.swift` (add sendRemoteMessage), `ConnectionManager.swift`

### 5. DoubleRatchetSession Persistence

Same pattern as PreKeyStore:
- Add `Codable` conformance to `DoubleRatchetSession` (serialize all state: root key, chain keys, skipped keys, counters, ratchet key pairs)
- `RemoteSessionManager` saves/loads sessions via `ChatDataEncryptor`
- Storage: `Documents/Security/sessions/{contactId}.enc`
- Load all persisted sessions on app launch
- Save after every encrypt/decrypt operation (debounced)

**Files:** Modify `DoubleRatchet.swift`, `RemoteSessionManager.swift`

### 6. TrustedContact mailboxId Integration

`TrustedContact` already has a `mailboxId` field. Wire it:
- Set `mailboxId` when accepting invite or receiving X3DH initial
- Use `mailboxId` to look up contacts from incoming messages
- Add `contactId(forMailboxId:)` lookup to `TrustedContactStore`

**Files:** Modify `TrustedContactStore.swift`

### 7. Worker Deploy

Code is ready. Run `npx wrangler deploy` to push v2 API to production.
